スノーフレークでスキーマを作成する手順：
Snowflake にログイン: Snowflake に管理者としてログインします。

ロールの確認: 必要な権限があるロール（例：SYSADMIN または ACCOUNTADMIN）を使用しているか確認します。USE ROLE SYSADMIN; で適切なロールを設定できます。


USE ROLE SYSADMIN;
データベースの選択: スキーマはデータベース内に作成されるため、まずスキーマを作成したいデータベースを選択します。以下のコマンドでデータベースを選択できます。


USE DATABASE <データベース名>;
スキーマの作成: 以下の SQL コマンドを実行して、新しいスキーマを作成します。


CREATE SCHEMA <スキーマ名>;
例えば、garden_plants というスキーマを作成したい場合は：


CREATE SCHEMA garden_plants;
スキーマ作成の確認: 作成したスキーマが存在するか確認するには、以下の SQL コマンドを使用してデータベース内のスキーマの一覧を確認できます。


SHOW SCHEMAS;
これで新しいスキーマが作成されます。

スキーマ作成時にオプションを設定する（任意）:
スキーマの所有者を指定: スキーマ作成時に所有者を指定することもできます。例えば、SYSADMIN ロールが所有するスキーマを作成する場合は、次のように指定します。


CREATE SCHEMA <スキーマ名> WITH OWNER = SYSADMIN;
データベースの指定: 新しいスキーマを特定のデータベース内に作成することもできます。


CREATE SCHEMA <データベース名>.<スキーマ名>;
これで、スキーマを作成する準備が整います！
