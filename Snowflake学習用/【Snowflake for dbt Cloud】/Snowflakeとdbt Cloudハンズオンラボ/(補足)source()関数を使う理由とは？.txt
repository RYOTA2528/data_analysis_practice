なぜ source() 関数を使うの？
1. 依存関係（リネージ：データの流れ）が明確になる
source() を使うと、dbt が「このモデルはこの生データを使っている」と理解できます。

そのため、データリネージ（系統図）で視覚的に確認できるようになります。

例：orders → stg_tpch_orders → fact_sales

2. ソースにもテスト・ドキュメントを適用できる
tpch_sources.yml に書いた内容に基づいて、ソースにも unique や not_null テストをかけられます。

また、dbt docs generate でドキュメント化もできます。

3. 環境の切り替えが柔軟になる
例えば、開発環境では dev_db、本番環境では prod_db を使いたい場合。

source() を使っておけば、YAMLファイルの database や schema を変えるだけで対応できます。

ハードコードしていたら、全モデルを1つ1つ修正する羽目に…

4. チーム開発でも安心
他の開発者がどのデータを使っているか、source() を使えばすぐにわかる。

モデル間の接続が明示的なので、属人性がなくなる。
