/*
【GCPにおけるガバナンスの仕組み】
主に以下の2つの方法を実施できます。

①Cloud Data Catalog
目的：Cloud Data Catalogは、Google Cloud内でのデータセットに関するメタデータを管理するツールです。このツールを使用すると、ユーザーがどのデータセットを検索できるか、またそのデータセットに関する詳細情報を簡単に把握することができます。
機能：
メタデータ管理：データセットのメタデータを検索可能にし、簡単にアクセスできます。
タグ付け：データセットにタグを付けて分類できます。例えば、特定のデータが機密性の高い情報を含む場合、その列にフラグを立てることができます。
ユーザーアクセスの統一管理：異なるユーザーが異なるアクセス権を持つ場合でも、統合されたインターフェースでデータセットを簡単に検出できます。


②Cloud Data Loss Prevention (DLP) API
目的：DLP APIは、データセット内の機密情報を自動的に検出し、秘匿化（データの匿名化やマスキング）するためのツールです。このAPIは、高速かつスケーラブルに動作し、セキュリティを強化します。
機能：
機密データの識別：クレジットカード番号、社会保障番号、電話番号など、機密データを迅速に識別し、マスキングします。
秘匿化：機密データを秘匿化（データを匿名化またはマスク）し、漏洩や不正アクセスから保護します。
規制遵守：個人情報保護法（GDPRなど）に準拠するために、機密データを適切に扱うためのツールとして役立ちます。
*/
